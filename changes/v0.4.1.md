# v0.4.1

## Highlights

OpenRouter is now a first-class provider. Use `--provider openrouter` with any of OpenRouter's hundreds of models (e.g. `--model openai/gpt-oss-20b:free`, `--model anthropic/claude-3.5-sonnet`). The new `OPENROUTER_API_KEY` env var slots into the existing provider list, and `yoke --provider openrouter` (no `--model`) lists the catalog with `name`, `created`, and `context_length` fields.

The nu tool gains `--config <file.nu>`, a startup script run once against the shared engine state. Statements like `use module-name`, `def`, `hide`, and env mutations persist across every nu tool call without the model having to re-import. Parse and eval errors are reported at startup with file path and span (mirroring nushell's own error format), so misconfigurations surface immediately rather than 50 turns in.

CLI gains `--thinking off|minimal|low|medium|high` for extended-thinking models (thanks @kiil for the PR #3) and `--max-turns <N|unlimited>` to cap or uncap agent loop iterations.

`ProviderError::RateLimited` now renders cleanly when the upstream provider sends no `Retry-After` header -- previously displayed as `Rate limited, retry after Nonems`, now `Rate limited (no retry-after)` (or `... retry after 2000ms` when present).

yoagent is now vendored into the repo rather than tracked as a submodule. This lets us iterate on yoke-specific changes without each one having to make sense for upstream, while still leaving the door open to contributing things back.
