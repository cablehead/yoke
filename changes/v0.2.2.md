# v0.2.2

## Highlights

**Embedded nushell tool.** yoke now embeds a full nushell engine as a tool (`nu`), available alongside `bash`. The LLM can execute nushell scripts with access to all nushell builtins and the parent environment. This is a 125-line tool backed by `nu-cli`, `nu-command`, and `nu-engine` -- the same engine that powers nushell itself. Select it with `--tools nu` or it's included in `--tools all`.

**Web search citations as structured metadata.** Both Anthropic and Gemini web search results now attach citation/source data to the assistant message's `metadata` field instead of dumping raw JSON into the text content. Anthropic citations include `title`, `url`, and `cited_text`. Gemini grounding includes `groundingChunks`, `groundingSupports`, and `webSearchQueries`. The web UI renders both as clickable source links in a styled footer.

**Web UI: provider/model picker.** The Datastar-powered UI now has a provider dropdown and a filterable model list. Selecting a provider fetches available models from the API, cached in the cross.stream store with a 6-hour TTL for instant filtering. Only providers with API keys set in the environment are shown. The model list renders as clickable divs with the selected model highlighted.

**Web UI: run persistence and history.** Completed runs are stored in the cross.stream event store. The `/runs` page lists past runs with prompt preview and model name. Clicking a run replays the full conversation as a stack of cards. Progressive rendering shows completed cards immediately while the current turn streams live.

**Web UI: progressive card stack.** The streaming view now renders completed turns as distinct cards (user prompt, assistant response, tool result) as they finish, with a live streaming placeholder at the bottom. On `agent_end`, the full conversation replaces the stream via `render-run`.

Anthropic `total_tokens` is now computed correctly (was previously 0).
