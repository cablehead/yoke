# v0.3.0

## Highlights

The builtin nushell tool now automatically converts output to nuon (Nushell Object Notation), so structured data round-trips cleanly -- multi-word string values like "Widget A" are properly quoted and parseable.

A new optional `input` parameter accepts JSON data that gets piped as `$in` to the nushell command. This lets LLMs pass structured data natively (as JSON) without dealing with string quoting inside the command. The tool description includes examples to guide models toward correct usage patterns.

Added an eval harness (`tests/tools/nu/`) for iterating on tool descriptions and behavior across providers. Each case is a markdown file with a prompt and evaluation criteria; `perform.nu` runs the case through yoke and checks the output.
