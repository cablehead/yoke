# v0.4.0

## Highlights

The nu tool now supports Nushell plugins and module include paths. Use `--plugin` to load plugin binaries (e.g. `nu_plugin_polars`) and `-I` / `--include-path` for module resolution. Plugin names are automatically included in the tool description so the LLM can discover commands via `help`.

The tool description now guides LLMs to use `help` liberally -- both for core nushell commands and for plugin subcommands. This significantly improves first-try success rates when working with plugins.

Tool definitions are emitted as a `{"type":"tools"}` JSONL line before `agent_start`, making it possible to inspect what the model sees.

Nushell dependencies updated from 0.111.0 to 0.112.1.
